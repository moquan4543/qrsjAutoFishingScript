# Once Human 自動釣魚腳本 (AutoHotkey v2.0)

本專案提供一個基於狀態機 (State Machine) 架構的全自動釣魚 AHK v2.0 腳本。包含 Windows Media OCR 咬鉤與 QTE 文字辨識、時間控制開路拉魚、魚種過濾篩選、解析度自動適應、防呆超時自適應恢復與局部區域優化。

## 專案結構
* **[AutoFishing.ahk](file:///C:/Users/moqua/Documents/AutoHotkey/OnceHumanAutoFishing/AutoFishing.ahk)** - 釣魚核心 AHK 腳本。
* **[OCR.ahk](file:///C:/Users/moqua/Documents/AutoHotkey/OnceHumanAutoFishing/OCR.ahk)** - Descolada OCR AHK v2 封裝庫 (依賴 Windows 10/11 內置 OCR 引擎)。
* **[crop_templates.py](file:///C:/Users/moqua/Documents/AutoHotkey/OnceHumanAutoFishing/crop_templates.py)** - 圖片特徵自動裁切 Python 腳本（用於基準解析度）。
* **Requirement.md** - 本專案的開發需求檔。
* **stateImages/cropped/** - 包含您手動去背後的自訂特徵模板：
  * `state3_indicator.png` - 下竿等待魚咬鉤 (去背綠色掛鉤圖示)。
  * `state8_indicator.png` - 釣上與收魚 (F鍵提示去背圖)。

## 快速開始

### 1. 安裝與準備
1. 確保已安裝 [AutoHotkey v2.0+](https://www.autohotkey.com/)。
2. 腳本已內含下載好的 [OCR.ahk](file:///C:/Users/moqua/Documents/AutoHotkey/OnceHumanAutoFishing/OCR.ahk)。請確保 Windows 系統的「語言與地區」設定中已安裝繁體中文語言包（以確保能識別中文字）。
3. 啟動遊戲《Once Human》，裝備好釣竿並站在水邊。

### 2. 啟動與操作熱鍵
雙擊執行 `AutoFishing.ahk` 啟動腳本。
* `F8`：啟動/恢復自動釣魚（自動拋竿開始循環，拋竿不強制拉回鼠標中心）。
* `F9`：暫停自動釣魚（會安全地釋放所有按鍵）。
* `F11`：切換是否挑選篩選魚種功能（預設開啟，在左上角 ToolTip 顯示狀態）。
* `F12`：強制終止腳本。

## 優化與技術細節說明

### 1. 智慧避障全視窗魚種篩選 (F11 挑選魚種)
* 為避免長按左鍵引起張力過大而斷線，且確保拉魚初期的 QTE 鍵盤響應，魚種過濾被設計為**非阻塞背景掃描**：
  * **按 F11 開關**：使用者可隨時切換是否挑選魚種，預設為開啟。
  * **避障全視窗掃描 (右側 75%)**：排除左側 25% 寬度區域，防止 OCR 誤讀到腳本自身渲染於左上角 `(10, 10)` 的狀態 ToolTip（該 ToolTip 含有 `"極寒水母/電鰻"` 等文字，若不排除會造成 100% 誤判放行）。
  * **高容許度單字匹配**：只要右側 75% 畫面內包含 `"極/极/寒/母"` 之一即認定為極寒水母；包含 `"電/电/鰻/鳗"` 之一即認定為電鰻。
    * **匹配成功**：保留左鍵長按，起步時間戳記自動補償，照常進行拉魚循環。
    * **匹配失敗**：立即釋放左鍵，按下 `ESC` 鍵拉斷線放棄，等待 3 秒動畫完成後直接切回狀態 1 重新下竿拋線，極大提升極品魚的取得效率！

### 2. 拋竿失敗/未下水防卡偵測 (State 5 -> State 1)
* 在狀態 5 (拉魚) 過程中，我們新增了**未成功拋竿防卡重置機制**：
  * 使用 `DetectTextOCR()` 偵測視窗上半部是否出現 `"長按拋竿"` (如 "長"、"按"、"拋"、"竿") 文字。
  * 只要匹配該特徵字元達 3 個 or 以上，即代表釣竿並未拋出 (可能拋竿被干擾或未拋成功)，此時腳本會**立即自動切換回狀態 1** 重新拋竿，避免在未拋竿狀態下空拉滑鼠。
  * 同樣，此防卡偵測也存在於狀態 3 (等待咬鉤) 之中。

### 3. 狀態 1 步法防踢防斷線 (Anti-AFK)
* 當進入狀態 1 (釣竿就緒) 且腳本未暫停 (`!IsPaused`) 時，會無縫執行微幅按鍵移動：
  * 按下 A 鍵微幅移動（左移 150ms），等待 200ms。
  * 按下 D 鍵微幅移動（右移 150ms），等待 500ms 動作收尾。
* 此機制僅使用 A/D 左右平移，避開了 W 鍵前移可能造成的角色位移，同時確保角色不會因為長時間未做出動作被判定 AFK 踢出。

### 4. 收魚與異常結算雙重偵測 (State 5 -> State 8)
* 在狀態 5 (拉魚) 過程中，除了偵測代表魚體力耗盡的 `"F 收起 G 放生"` 提示之外，額外新增了**捕獲結算畫面防卡偵測**：
  * 偵測文字擴展到 `"重量"` (如 "重" + "量") 或 `"尺寸"` (如 "尺" + "寸")。
  * 若拉魚過程中因延遲或判定遺漏，導致畫面已彈出魚隻細節結算頁面（含有重量、尺寸資訊），腳本會立刻識別並**自動跳轉到狀態 8** 開始收魚，確保時序不卡在狀態 5。

### 5. 收魚保障確認機制 (State 8)
* 為了防止網路延遲、伺服器丟包導致 F 鍵未響應，在狀態 8 新增了**收魚確認迴圈**：
  * 等待 6 秒動畫結束後按下 `F` 鍵並等待 1.5 秒。
  * 重新使用 OCR 識別視窗中央是否仍殘留 `"F 收 起 G 放 生"` 相關字樣。
  * 若仍有文字，代表 `F` 鍵未成功響應，腳本會**重複按壓 F 鍵並進行 OCR 檢測**，直到文字完全消失才回到狀態 1，確保裝備不丟包。
  * **超時安全閥**：若重試 6 次 (約 9 秒) 後文字仍未消失，會自動觸發狀態重置。

### 6. 智慧狀態偵測與防呆自適應恢復 (AFK State Recovery)
* 當腳本在某個狀態停留超過 60 秒（例如網路延遲時），將觸發**智慧狀態檢測**：
  * 釋放所有按鍵，暫停 UI 狀態更新，並執行全視窗 OCR 掃描。
  * **卡在狀態 08 (收魚畫面)**：若檢測到 `"收起"`、`"放生"` 或 `"F"` + `"收"`，自動切入狀態 **08**。
  * **卡在狀態 03 (等待魚咬鉤)**：若檢測到 `"收線"`，判定釣線已在水中，自動切入狀態 **03** 繼續等待，避免收竿重拋浪費魚餌。
  * **卡在狀態 01 (釣竿就緒)**：若皆未辨識到，則設為 **01**。
  * 倒數 5 秒後**自動切換至該狀態繼續執行**，不需 any 手動彈窗操作。
