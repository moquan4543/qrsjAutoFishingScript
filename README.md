# Once Human 自動釣魚腳本 (AutoHotkey v2.0)

本專案提供一個基於狀態機 (State Machine) 架構的全自動釣魚 AHK v2.0 腳本。支援遊戲 **3.0.2 最新版本 QTE 機制**、Windows Media OCR 咬鉤與 QTE 文字辨識、時間控制開路拉魚、3.0 秒黃金視窗魚種過濾篩選、解析度自動適應、防呆超時自適應恢復與極速 QTE 偵測優化。

## 專案結構
* **[AutoFishing.ahk](file:///C:/Users/moqua/Documents/AutoHotkey/OnceHumanAutoFishing/AutoFishing.ahk)** - 釣魚核心 AHK 腳本。
* **[OCR.ahk](file:///C:/Users/moqua/Documents/AutoHotkey/OnceHumanAutoFishing/OCR.ahk)** - Descolada OCR AHK v2 封裝庫 (依賴 Windows 10/11 內置 OCR 引擎)。
* **[crop_templates.py](file:///C:/Users/moqua/Documents/AutoHotkey/OnceHumanAutoFishing/crop_templates.py)** - 圖片特徵自動裁切 Python 腳本（用於基準解析度）。
* **Requirement.md** - 本專案的開發需求檔。
* **stateImages/cropped/** - 包含您手動去背後的自訂特徵模板：
  * `state3_indicator.png` - 下竿等待魚咬鉤 (去背綠色掛鉤圖示)。
  * `state8_indicator.png` - 釣上與收魚 (F鍵提示去背圖)。

## 快速開始

### 1. 安裝與準備
1. 確保已安裝 [AutoHotkey v2.0+](https://www.autohotkey.com/)。
2. 腳本已內含下載好的 [OCR.ahk](file:///C:/Users/moqua/Documents/AutoHotkey/OnceHumanAutoFishing/OCR.ahk)。請確保 Windows 系統的「語言與地區」設定中已安裝繁體中文語言包（以確保能識別中文字）。
3. 啟動遊戲《Once Human》，裝備好釣竿並站在水邊。

### 2. 啟動與操作熱鍵
雙擊執行 `AutoFishing.ahk` 啟動腳本。
* `F8`：啟動/恢復自動釣魚（自動拋竿開始循環，拋竿不強制拉回鼠標中心）。
* `F9`：暫停自動釣魚（會安全地釋放所有按鍵）。
* `F11`：切換是否挑選篩選魚種功能（預設開啟，在左上角 ToolTip 顯示狀態）。
* `F12`：強制終止腳本。

## 優化與技術細節說明

### 1. 遊戲 3.0.2 版本 QTE 機制支援 (極速 ~30ms 響應)
* **連續點擊 (Rapid Click)**：
  - 自動辨識 `連續點擊` 文字與方向（A 或 D）。
  - 自動觸發 7 次高速連點 (Down 35ms, Up 45ms)，快速消除 3~6 個點擊圖示並恢復拉魚。
* **持續長按 (Hold)**：
  - 自動辨識 `持續長按` 文字與方向（A 或 D）。
  - 固定持續長按 3.5 秒後切回拉魚狀態。
* **極速 QTE 偵測頻率 (30ms)**：
  - 將 QTE 區域偵測獨立為每 30ms 掃描一次，反應延遲低至 ~30ms，完美適應 3.0.2 版變短的 QTE 時限。

### 2. 3.0 秒黃金視窗智慧魚種過濾 (F11 挑選魚種)
* **3.0 秒嚴格時間視窗**：
  - 魚種篩選鎖定在首次拉魚的前 3.0 秒 (3000ms) 內進行高頻辨識（採用中央 50% 區域 OCR 掃描）。
  - 只要 3.0 秒內讀取到包含 `"極/极/寒/母"` (極寒水母) 或 `"電/电/鰻/鳗"` (電鰻) 之一，立刻通過篩選並正常拉魚。
  - 若滿 3.0 秒仍未匹配到目標魚，立刻釋放按鍵並按下 `ESC` 斷線，絕不拖延時間，也保證不會讓雜魚 (如沙丁魚) 被拉上來。

### 3. 拋竿失敗/未下水防卡偵測 (State 5 -> State 1)
* 在狀態 5 (拉魚) 過程中，新增了**未成功拋竿防卡重置機制**：
  * 使用 `DetectTextOCR()` 偵測視窗上半部是否出現 `"長按拋竿"` (如 "長"、"按"、"拋"、"竿") 文字。
  * 只要匹配該特徵字元達 3 個 or 以上，即代表釣竿並未拋出，腳本會**立即自動切換回狀態 1** 重新拋竿。

### 4. 狀態 1 步法防踢防斷線 (Anti-AFK)
* 當進入狀態 1 (釣竿就緒) 且腳本未暫停 (`!IsPaused`) 時，會無縫執行微幅按鍵移動：
  * 按下 A 鍵微幅移動（左移 150ms），等待 200ms。
  * 按下 D 鍵微幅移動（右移 150ms），等待 500ms 動作收尾。

### 5. 收魚與異常結算雙重偵測 (State 5 -> State 8)
* 在狀態 5 (拉魚) 過程中，除了偵測 `"F 收起 G 放生"` 提示之外，新增了**捕獲結算畫面防卡偵測**：
  * 偵測文字擴展到 `"重量"` (如 "重" + "量") 或 `"尺寸"` (如 "尺" + "寸")。
  * 若拉魚過程中畫面已彈出魚隻細節結算頁面，腳本會立刻識別並**自動跳轉到狀態 8** 開始收魚。

### 6. 收魚保障確認機制 (State 8)
* 為了防止網路延遲導致 F 鍵未響應，新增**收魚確認迴圈**：
  * 等待 6 秒動畫結束後按下 `F` 鍵並等待 1.5 秒。
  * 重新識別視窗中央是否仍殘留收魚字樣，若仍有文字則**重複按壓 F 鍵並檢測**，直到文字完全消失。
